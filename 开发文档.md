```

├── index.html          # 主页 - 个人简介、技能展示、联系方式

│   └── main.js         # 交互逻辑 - 导航、筛选、动画等
├── blog/
│   └── learning-js.md  # 示例文章 - Markdown格式的博客内容

├── README.md           # 项目说明 - 使用方法和部署指南
└── 开发文档.md         # 本文档 - 开发过程和实现细节
```

## 3. 核心模块实现思路

### 导航栏模块
```javascript
// 移动端汉堡菜单切换
hamburger.addEventListener('click', () => {
    navLinks.classList.toggle('active');
});
```
- 使用position: sticky实现固定导航
- 移动端使用JavaScript控制显示/隐藏
- 添加背景模糊效果（backdrop-filter）

### 卡片组件
```css
.card {
    background: white;
    border-radius: 16px;
    box-shadow: 0 4px 15px rgba(0,0,0,0.1);
    transition: transform 0.3s ease;
}
.card:hover {
    transform: translateY(-10px);
}
```
- 统一的卡片样式（圆角、阴影、悬停效果）
- 使用CSS Grid实现响应式网格布局
- 通过data属性实现分类筛选

### 筛选功能
```javascript
filterBtns.forEach(btn => {
    btn.addEventListener('click', function() {
        const filter = this.dataset.filter;
        cards.forEach(card => {
            if (filter === 'all' || card.dataset.category === filter) {
                card.style.display = 'block';
            }
        });
    });
});
```
- 使用data-*属性标记分类
- JavaScript控制显示/隐藏
- 添加过渡动画效果

### Markdown渲染
```javascript
if (typeof marked !== 'undefined') {
    markdownContent.innerHTML = marked.parse(markdownText);
}
```
- 使用marked.js库解析Markdown
- 支持代码高亮
- 支持列表、引用等格式

### 滚动动画
```javascript
const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            entry.target.classList.add('visible');
        }
    });
});
```
- 使用IntersectionObserver检测元素进入视口
- 添加fade-in类控制动画
- 支持多种动画效果

## 4. 开发难点和注意事项

### 难点

1. **响应式布局**
   - 不同屏幕尺寸的适配
   - 移动端导航的处理
   - 网格布局的自适应

2. **动画性能**
   - 避免频繁触发重排
   - 使用transform代替top/left
   - 合理使用will-change

3. **Markdown解析**
   - marked.js库的加载时机
   - 代码块的样式处理
   - 特殊字符的转义

### 注意事项

1. **代码规范**
   - 使用一致的缩进（2空格）
   - CSS变量命名规范
   - JavaScript命名规范

2. **性能优化**
   - 图片懒加载
   - CSS/JS文件压缩
   - 使用CDN加载外部库

3. **兼容性**
   - 使用Autoprefixer处理CSS前缀
   - 测试主流浏览器
   - 处理旧版浏览器降级

4. **可维护性**
   - 代码注释清晰
   - 模块化设计
   - 易于扩展的结构

5. **部署准备**
   - 文件路径使用相对路径
   - 测试本地预览
   - 检查资源加载
